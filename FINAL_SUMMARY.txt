HOW IT WORKS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

SYSTEM FLOW
═══════════════════════════════════════════════════════════════════════════

1. SHARED TRADING KPIs (Redis)
   ↓
   Aggregates: Avg R, Win Rate, Execution Rate, Latency, Freshness
   Updates every 60s with full funnel state

2. WORK FREEZE DETECTION
   ↓
   IF: seededToExecuted < 40% OR fresh < 50% OR latency > 300s
   THEN: freezeNonCriticalWork = true
   
3. EXECUTION AGENT (Autonomous)
   ↓
   Monitors KPIs → Detects critical incidents
   Auto-creates CRITICAL engineering tasks when:
     • Latency > 300s
     • Fresh signals < 50%
     • Execution conversion < 40%
     • Stale signals > 50%
   
4. PRIORITY ENGINE (Task Ranking)
   ↓
   Scores all tasks by trading impact:
     EXECUTION: 10/10
     RISK: 9/10
     PERFORMANCE: 7/10
     CRITICAL_ENG: 7/10
     FEATURE: 4/10
     OPTIMIZATION: 3/10
     COSMETIC: 0/10
   
   During freeze: Only EXECUTION, RISK, PERFORMANCE allowed
   
5. EM ORCHESTRATION (Enhanced)
   ↓
   Selects next executable task (highest non-frozen score)
   Generates recommendations based on KPI violations
   Routes to correct execution pipeline
   
6. TASK EXECUTION WITH METRICS
   ↓
   Before execution: Record beforeMetrics
   After execution: Record afterMetrics
   Status: SUCCESS only if R improved (with evidence)
   
7. KPI FEEDBACK LOOP
   ↓
   Updates shared KPIs → Next cycle uses fresh data

EXAMPLE: EXECUTION CRISIS & RECOVERY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

SCENARIO: Seeded → Executed conversion crashes from 68% to 25%

09:00 → Trading KPIs update: seededToExecuted = 25%
09:01 → Work freeze triggers: "Execution rate critical"
09:01 → Execution agent detects incident (CRITICAL)
09:02 → Auto-creates task: "Fix execution conversion 25% < 40%"
09:02 → Priority engine ranks this task: 94/100 (highest)
09:02 → Task marked EXECUTABLE (not frozen)
09:03 → All other tasks except EXECUTION/RISK/CRITICAL frozen
09:03 → EM selects this task as next priority
09:04 → Engineer starts on task: "Fix seeded order blocking"
09:30 → Task completed with evidence:
        Before: seededToExecuted = 25%, latency = 450s
        After: seededToExecuted = 68%, latency = 85s
        Status: SUCCESS (metrics improved 🎯)
09:31 → Trading KPIs update: fresh → 88%, latency → 85s
09:31 → Work freeze automatically lifts (execution > 40%)
09:31 → Normal task permissions restored
09:31 → Morning metrics show: "+55% execution improvement"

RESULT: Coordinated, autonomous response. No manual intervention needed.
        Metrics tracked. Impact verified. System recovered.

KEY FEATURES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

✅ AGENT KPI SCORING
   Each agent has:
     - Functional score (core job)
     - Trading score (R impact) ← 60% WEIGHT
     - Penalty score (degradation)
   Total = fs×0.25 + ts×0.60 - ps×0.15
   
   Example Execution Agent:
     Functional: 8/10 (good engineering)
     Trading: 7/10 (good R impact)
     Penalty: 1/10 (minor issues)
     Total: 7.6/10 = GOOD status ✅

✅ WORK FREEZE (Automatic)
   When funnel fails:
     Only: EXECUTION, RISK, PERFORMANCE, CRITICAL_ENGINEERING
     Blocked: FEATURE, OPTIMIZATION, COSMETIC
   
   Freeze lifts automatically when metrics recover

✅ EXECUTION AGENT (Autonomous)
   Monitors:
     - Latency < 60s (target)
     - Fresh signals > 80% (target)
     - Seeded → executed > 60% (target)
     - Zero duplicate seeds (target)
   
   Auto-creates CRITICAL tasks on threshold breach

✅ PRIORITY ENGINE (Trading Impact)
   Ranks tasks by formula:
     score = severity × urgency × tradingImpact × confidence
   
   Example ranking during crisis:
     94/100 Fix execution conversion
     88/100 Fix stale signal reseeding
     75/100 Fix position mismatch
      5/100 ⛔ FROZEN: Implement dark mode
      0/100 ⛔ FROZEN: Refactor API

✅ TASK IMPACT TRACKING
   Every task captures:
     expectedRImpact: "+0.5R to +2R/day"
     beforeMetrics: { seededToExecuted: 12%, latency: 450s }
     afterMetrics: { seededToExecuted: 67%, latency: 85s }
     completionQuality: "SUCCESS" (only if improved)

✅ MORNING BRIEF (Performance-First)
   Instead of "Build passing", now shows:
     📊 Avg R: 0.35 (↑ +0.15)
     📊 Execution: 68% (↑ +8%)
     💡 HIGHEST ROI FIXES: +1.2R/day opportunity
